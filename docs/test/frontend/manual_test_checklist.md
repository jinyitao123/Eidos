# 前端手动测试清单

## 测试环境准备

1. 启动 Docker Compose (`docker compose up -d`)
2. 确认服务状态：Weave API (:8080)、MCP Server (:9091)、Neo4j (:7474)、PG (:5432)
3. 访问 http://localhost:8089 (Docker) 或 http://localhost:5180 (dev)

---

## Page 1: 项目列表 (ProjectList)

### 基础功能
- [ ] 页面加载后显示项目列表（至少有"备件管理"项目）
- [ ] 每个项目卡片显示：名称、描述、状态标签、统计数据（类/关系/规则/操作数）
- [ ] 点击"新建项目"按钮 → 弹出创建对话框
- [ ] 输入项目名称和描述 → 创建成功 → 列表刷新
- [ ] 点击删除按钮 → 弹出确认对话框 → 确认后删除 → 列表刷新
- [ ] 点击项目卡片 → 导航到 AgentBuild 页面

### 共享类检测
- [ ] 当存在多个项目时，如果有 `imported_from` 标记的类，显示"共享类"标签
- [ ] 项目卡片上显示跨项目共享的类名称

---

## Page 2: Agent 构建 (AgentBuild)

### 基础功能
- [ ] 页面显示 4 个阶段标签：场景分析、本体结构、规则动作、审查报告
- [ ] 默认选中第一个未完成的阶段
- [ ] 聊天输入框可输入消息
- [ ] 发送消息 → 显示用户消息气泡
- [ ] Agent 响应 → 显示 Agent 消息气泡（有思考动画）
- [ ] 消息区域自动滚动到底部

### 阶段流转
- [ ] 阶段 0 (场景分析) → Agent 响应后出现"确认并继续"按钮
- [ ] 点击确认 → 调用 `save_output` → 进入阶段 1
- [ ] 每个阶段使用独立的 session_id
- [ ] 已完成阶段的标签显示勾号图标
- [ ] 点击已完成的阶段可查看历史对话

### 文件上传
- [ ] 点击上传按钮 → 选择文件 → 文件内容作为消息发送
- [ ] 上传后输入框自动清空文件选择

### 回退
- [ ] 点击已完成阶段的"重做"按钮 → 弹出确认框
- [ ] 确认回退 → 清除该阶段及之后的输出 → 回到该阶段

### 错误处理
- [ ] Weave API 不可用时 → 显示错误提示
- [ ] 未获取到 JWT token → 显示认证错误

---

## Page 3: 图谱审查 (GraphReview)

### Schema 视图
- [ ] 页面加载后显示力导向图（类为节点，关系为边）
- [ ] 节点显示类名称
- [ ] 边显示关系名称和基数
- [ ] 鼠标拖拽节点 → 节点跟随移动，力导向重新计算
- [ ] 点击节点 → 显示类详情面板（属性列表）
- [ ] first_citizen 节点有特殊样式（颜色/大小）

### 实例视图
- [ ] 切换到"实例"标签
- [ ] 加载 Neo4j 数据 → 显示实例节点和边
- [ ] 显示统计信息（总节点数、总关系数、按标签分类）
- [ ] 不同类型的节点用不同颜色区分
- [ ] 点击节点 → 显示节点属性面板
- [ ] 节点可拖拽移动
- [ ] 无数据时显示"暂无实例数据"提示

### 拖拽连线 (Shift+Drag)
- [ ] 按住 Shift + 从节点 A 拖拽到节点 B
- [ ] 拖拽过程中显示虚线连线
- [ ] 松开鼠标 → 弹出创建关系的表单
- [ ] 填写关系信息 → 确认 → 关系添加到图上

### AI 辅助编辑
- [ ] 右下角浮动按钮 → 点击打开 AI 助手面板
- [ ] 可输入自然语言描述修改意图
- [ ] AI 返回建议 → 显示修改预览

---

## Page 4: 类编辑器 (ClassEditor)

### 基础功能
- [ ] 左侧显示类列表
- [ ] 点击类名 → 右侧显示属性表格
- [ ] 属性表格列：ID、名称、类型、必填、唯一、图同步、操作
- [ ] 点击属性行 → 内联编辑
- [ ] 添加新属性按钮 → 弹出属性表单
- [ ] 删除属性 → 确认对话框 → 删除
- [ ] 修改后显示"保存"和"取消"按钮
- [ ] 保存 → 调用 MCP 更新 YAML → 成功提示

### 属性类型特殊处理
- [ ] 选择 enum 类型 → 显示 enum_values 输入框
- [ ] 选择 derived → 显示公式输入框，隐藏 required 选项
- [ ] 选择 graph_sync → 勾选框
- [ ] 选择 configurable → 勾选框

### 拖拽排序
- [ ] 属性行左侧显示拖拽手柄 (⋮⋮)
- [ ] 拖拽属性行 → 调整顺序
- [ ] 拖拽过程中目标位置有视觉提示（高亮/间隔）
- [ ] 松开后属性列表更新顺序
- [ ] 保存后顺序持久化

### AI 辅助编辑
- [ ] 浮动 AI 按钮 → 打开建议面板
- [ ] 可输入"建议添加什么属性"等自然语言
- [ ] AI 返回属性建议列表

### 关系标签页
- [ ] 切换到"关系"标签 → 显示该类的出向关系
- [ ] 切换到"被引用"标签 → 显示该类的入向关系

---

## Page 5: 规则编辑器 (RuleEditor)

### 规则标签
- [ ] 显示所有规则列表
- [ ] 每条规则显示：ID、名称、严重度标签、触发类型、条件、操作
- [ ] 严重度标签颜色：critical=红色, warning=黄色, info=蓝色
- [ ] 点击规则 → 展开编辑面板
- [ ] 编辑规则参数 → 保存

### 操作标签
- [ ] 切换到"操作"标签
- [ ] 显示所有 action 列表
- [ ] 每个 action 显示：ID、名称、参数、写入目标、触发规则、权限、决策日志标记
- [ ] decision_log=true 的 action 有特殊标识

---

## Page 6: 审查报告 (ReviewReport)

### 基础功能
- [ ] 页面自动运行验证（调用 validate_yaml）
- [ ] 显示三类问题：一致性(C)、完整性(P)、建议(O)
- [ ] 一致性问题标红，阻止发布
- [ ] 完整性和建议问题标黄/蓝，不阻止
- [ ] 显示通过项（绿色勾号）
- [ ] 顶部汇总：X 个问题，Y 个通过

### 修复建议
- [ ] 部分问题显示"修复"按钮
- [ ] 点击修复 → 自动应用修复 → 问题消失
- [ ] 修复后重新验证

---

## Page 7: 发布管道 (PublishPipeline)

### 基础功能
- [ ] 显示 7 个步骤卡片，初始状态为"待执行"
- [ ] 每个步骤显示预估信息（表数量、工具数量等）
- [ ] 点击"执行管道"按钮
- [ ] 步骤依次变为"执行中" → "完成"（有进度动画）
- [ ] 全部完成后显示成功提示

### 文件预览
- [ ] 完成的步骤显示"查看文件"按钮
- [ ] 点击 → 弹出文件预览模态框
- [ ] 多文件时显示标签页切换
- [ ] 文件内容显示文件名和大小
- [ ] 代码内容有语法着色（SQL、TypeScript、YAML、Go）

### Pipeline 日志
- [ ] 底部可展开"执行日志"区域
- [ ] 显示 pipeline 的标准输出

### 错误处理
- [ ] Pipeline 失败 → 对应步骤显示红色错误状态
- [ ] 错误信息显示在步骤卡片下方
- [ ] 可点击"重试"重新执行

---

## 跨页面功能

### 导航
- [ ] 侧边栏/顶部导航可在 7 个页面间切换
- [ ] 当前页面高亮
- [ ] 项目上下文在页面间保持

### 数据一致性
- [ ] AgentBuild 保存的 YAML → GraphReview 页面正确渲染
- [ ] ClassEditor 修改属性 → GraphReview 反映变化
- [ ] RuleEditor 修改规则 → ReviewReport 重新验证
- [ ] ReviewReport 通过 → PublishPipeline 可执行

---

## 性能与边界

- [ ] 大本体（20+ 类、50+ 属性）→ 图谱渲染流畅
- [ ] 快速切换页面 → 无白屏/闪烁
- [ ] 网络断开 → 合理的错误提示（不是空白页）
- [ ] 并发操作（快速双击按钮）→ 不产生重复请求
