# 本体工具 · 页面设计与功能规格

**inocube 智能平台 · 本体工具（基于 Weave Agent 平台）**

v1.0 · 2026.03

---

> *七个页面，一条完整的本体开发工作流：*
> *项目列表 → Agent构建对话 → 图谱审核 → 类属性编辑 → 规则编辑 → 审核报告 → 发布管道*

---

## 全局设计规范

### 架构定位

本体工具是一个**独立部署的 Web 应用**，嫁接在 Weave Agent 平台之上。它复用 Weave 的 Agent 运行时（通过 `/api/` 代理调用 Weave API :8080）、认证体系（Bearer Token）和设计语言（Loom），但拥有自己的前端部署、布局和导航——与备件管理应用的架构模式完全一致。

```
┌─────────────────────────────────────────────────────────────┐
│  本体工具（独立 SPA，端口 :5180 / Docker :8089）              │
│                                                              │
│  ┌─ TopBar ──────────────────────────────────────────────┐  │
│  │  ✦ inocube · 本体工具                    用户 · 角色   │  │
│  └────────────────────────────────────────────────────────┘  │
│  ┌─ SideNav ─┬─ 内容区 ──────────────────────────────────┐  │
│  │            │                                           │  │
│  │  项目列表  │        页面内容                            │  │
│  │  ────────  │                                           │  │
│  │  构建对话  │    最大内容宽度 960px，水平居中             │  │
│  │  图谱审核  │    两侧自然留白                            │  │
│  │  规则编辑  │                                           │  │
│  │  审核报告  │                                           │  │
│  │  发布管道  │                                           │  │
│  │            │                                           │  │
│  └────────────┴───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

#### 与 Weave 平台的关系

```
本体工具前端 (:5180)              Weave API (:8080)
  │                                  │
  ├─ /api/* ─── nginx 代理 ────────→ │  Agent 对话、Session、认证
  │                                  │
  ├─ /mcp/* ─── nginx 代理 ────→ 本体 MCP Server (:9091)
  │                                  │  10 个本体工具 + 6 个图谱查询工具
  │                                  │
  └─ 静态资源 ── nginx 直接服务       └─ 调度 Agent 时自动调用 MCP Server
```

本体工具不修改 Weave Console 的任何代码。Weave Console (:3000) 是开发者工具（管理 Agent、查看运行记录），本体工具是业务工具（设计和发布本体定义）。两者独立部署，共享同一个 Weave API 后端。

### 布局

内容区最大宽度 960px（比备件管理应用的 720px 宽，因为本体编辑需要更多信息密度）。

**TopBar：** 左侧品牌标识「✦ inocube · 本体工具」，右侧用户信息和角色切换。

**SideNav：** 左侧固定导航栏。顶层显示项目列表入口；进入某个项目后，导航切换为该项目的子页面（构建对话、图谱审核、规则编辑、审核报告、发布管道）。

### 导航层级

```
项目列表（首页）
  └── 某个本体项目
        ├── Agent 构建对话
        ├── 图谱审核
        │     ├── 结构视图（schema 图谱）
        │     ├── 实例视图（实例图谱）
        │     └── 类属性编辑（点击节点进入）
        ├── 规则编辑（SideNav 入口）
        ├── 审核报告
        └── 发布管道
```

进入某个本体项目后，TopBar 显示项目名称和版本，SideNav 显示该项目的子页面列表。

### 通用交互

- 所有列表页的卡片点击后进入详情页，详情页左上角有"← 返回"
- 表格类数据支持行内编辑（点击单元格变为输入框）
- Agent 辅助编辑按钮统一放在页面右下角或操作栏末尾
- 所有破坏性操作（删除类、删除属性、回滚版本）需二次确认
- "客户可调"参数统一用绿色小标签标注

### 技术栈

| 层 | 技术选型 | 说明 |
|---|---|---|
| 框架 | React 19 + TypeScript | 与备件管理应用一致 |
| 构建 | Vite | 开发代理 + 生产构建 |
| 样式 | CSS Modules + Loom 设计语言 | 复用 Weave 的配色、字体、间距体系 |
| 图谱渲染 | d3-force | 力导向布局（schema 图谱 + 实例图谱） |
| 状态管理 | React Context | Agent 状态、项目上下文 |
| API 通信 | fetch + Weave API (/api/*) | Agent 对话、Session 管理 |
| MCP 通信 | fetch + JSON-RPC 2.0 (/mcp/*) | 本体工具调用 |
| 部署 | nginx (Docker) | 静态资源 + /api/ 代理到 Weave + /mcp/ 代理到本体 MCP |

## 页面二：Agent 构建对话

### 定位

新建本体后的第一个页面。用户上传调研文档，四个Agent依次协作输出本体初稿。

### 页面结构

```
┌──────────────────────────────────────────────────────────┐
│  ← 返回  设备运维 · 本体构建        [场景分析][架构][规则][审核]│
├──────────────────────────────────────────────────────────┤
│                                                          │
│  对话流区域（纵向滚动）                                   │
│                                                          │
│  [用户消息] 上传调研文档                                  │
│                                                          │
│  [S1 场景分析师] 分析结果卡片：                           │
│    · 第一公民判断 + 理由                                  │
│    · 核心类列表（彩色标签）                               │
│    · 关系列表（紫色标签）                                 │
│    · 共享类检测                                          │
│    · [调整第一公民] [补充类] [确认，下一步]               │
│                                                          │
│  [S2 本体架构师] 类和属性设计：                           │
│    · 每个类的属性统计卡片                                 │
│    · [查看完整YAML] [图谱预览] [确认，下一步]            │
│                                                          │
│  [S3 规则设计师] 正在工作...                              │
│                                                          │
│  ──────────────────────────────────────────              │
│  [输入框: 跟构建Agent对话]                    [发送]      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 功能清单

| # | 功能 | 说明 |
|---|---|---|
| 2.1 | 构建进度条 | 顶部四段进度条：场景分析 → 本体架构 → 规则设计 → 审核。已完成的段绿色，当前段蓝色，未到的段灰色。下方标注每段名称 |
| 2.2 | 调研文档上传 | 用户可上传 .md / .docx / .txt 格式的调研文档。上传后自动触发场景分析师Agent |
| 2.3 | Agent角色区分 | 四个Agent用不同颜色头像区分：S1（赭陶色）、S2（绿色）、S3（紫色）、S4（灰色）。每条消息显示Agent名称和角色ID |
| 2.4 | 结构化输出卡片 | Agent的输出不是纯文字，包含结构化卡片：第一公民判断卡片、核心类列表（彩色标签）、关系列表、属性统计、审核结果 |
| 2.5 | 阶段确认机制 | 每个Agent完成后，底部显示操作按钮：调整按钮（修改Agent的输出）+ 确认按钮（进入下一阶段）。用户必须确认后才进入下一个Agent |
| 2.6 | 人工干预对话 | 用户随时可在底部输入框发消息干预。如："给维修工单加一个紧急程度属性"→ 当前活跃的Agent响应并修改输出 |
| 2.7 | 图谱预览 | 本体架构师完成后，可点击"图谱预览"在对话流中渲染一张简化的图谱视图（节点+边，不可编辑） |
| 2.8 | YAML预览 | 点击"查看完整YAML"在对话流中展开一个代码块，展示当前生成的完整YAML |
| 2.9 | 共享类自动检测 | 场景分析师检测到当前项目的类名与其他已发布本体的类名匹配时，自动提示"此类可从备件管理本体导入复用" |
| 2.10 | 阶段回退 | 点击顶部进度条中已完成的阶段名可回退。回退后该阶段及后续阶段的输出清空，从该阶段重新开始 |

### 四个Agent的输入输出

| Agent | 输入 | 输出 | 操作按钮 |
|---|---|---|---|
| S1 场景分析师 | 调研文档全文 | 第一公民判断+理由、核心类列表、关系列表、数据源清单、已知约束 | [调整第一公民] [补充类] [确认→下一步] |
| S2 本体架构师 | S1的输出 | 完整的类定义（每个类的属性列表、类型、是否派生、是否图谱同步）、关系定义（方向、多重性、边属性） | [查看YAML] [图谱预览] [确认→下一步] |
| S3 规则设计师 | S1+S2的输出 | 规则定义（触发、条件、动作、参数、是否客户可调）、受控动作定义（参数、写回逻辑、触发规则、权限） | [确认→下一步] |
| S4 审核员 | S1+S2+S3的完整YAML | 审核报告（一致性、完整性、冗余、命名）、每条问题的修正建议 | [自动修复全部] [逐条处理] [进入可视化审核] |

### 数据流

```
用户上传文档
  ↓ 发送给Weave Agent（S1）
S1输出 → 用户确认
  ↓ S1输出作为S2的输入
S2输出 → 用户确认
  ↓ S1+S2输出作为S3的输入
S3输出 → 用户确认
  ↓ 完整YAML作为S4的输入
S4输出审核报告
  ↓ 用户选择"进入可视化审核"
跳转到 → 图谱审核页
```

## 页面三：图谱全貌审核

> 视觉设计稿：`design/ontology_graph_review.svg`

### 定位

Agent构建完成后（或进入已发布本体时），展示本体的全局关系图谱。这是人审核本体结构的主界面。

### 页面结构

```
┌──────────────────────────────────────────────────────────┐
│  ← 返回  备件管理 · 本体审核           [审核报告] [发布]   │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│  类列表   │           交互式图谱区域                       │
│          │                                               │
│ ★ 库存头寸│    节点 = 类（可拖拽、可点击）                │
│   备件    │    边 = 关系（可点击查看详情）                │
│   库房    │                                               │
│   设备    │    第一公民节点放大高亮                        │
│   出入库  │    颜色区分：                                 │
│   采购单  │    · 赭陶色 = 第一公民                        │
│   快照    │    · 绿色 = 核心类                            │
│          │    · 紫色 = 事件/快照类                        │
│ ──────── │                                               │
│  规则(8)  │    节点上显示：类名 + 属性数量                 │
│  动作(8)  │    边上显示：关系名                           │
│ ──────── │                                               │
│ + 新增类  │    底部图例栏                                 │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘
```

### 功能清单

| # | 功能 | 说明 |
|---|---|---|
| 3.1 | 交互式图谱渲染 | 使用力导向布局（d3-force或类似库）渲染节点和边。节点可拖拽调整位置，双指缩放，拖拽画布平移 |
| 3.2 | 节点样式 | 圆角矩形。第一公民：赭陶色填充、更大尺寸、粗边框。核心类：绿色填充。事件/快照类：紫色填充。每个节点显示类名（14px粗体）和属性数量（12px） |
| 3.3 | 边样式 | 实线箭头。边上标注关系名。自引用关系（如备件→可替代→备件）用虚线弧线。边的颜色比节点浅一档 |
| 3.4 | 点击节点 | 点击节点进入"类属性编辑"页面（页面四） |
| 3.5 | 点击边 | 点击边弹出浮层，展示：关系名、方向、多重性、边属性列表。可在浮层中直接编辑边属性 |
| 3.6 | 左侧类列表 | 所有类名列表，第一公民用★标记。点击类名在图谱中高亮对应节点并居中。底部"+ 新增类"按钮 |
| 3.7 | 左侧规则/动作入口 | 类列表下方分隔线后，显示"规则(8)"和"动作(8)"链接。点击进入规则编辑页（页面五） |
| 3.8 | 新增类 | 点击"+ 新增类"→ 弹出面板输入类名和描述 → 图谱中添加新节点（初始无关系，需手动拖拽连线或在属性编辑页添加关系） |
| 3.9 | 删除类 | 右键节点 → 删除（需二次确认）。删除时检查：如果有关系引用此类，提示先删除关系 |
| 3.10 | 图例栏 | 图谱底部固定一行图例：赭陶色=第一公民、绿色=核心类、紫色=事件/快照类、实线=结构关系、虚线=自引用 |
| 3.11 | 审核报告入口 | 右上角"审核报告"按钮，点击进入审核报告页（页面六） |
| 3.12 | 发布入口 | 右上角"发布"按钮。如果有未处理的审核问题，按钮置灰并提示"请先处理审核报告中的问题" |
| 3.13 | Agent辅助 | 图谱区域右下角浮动按钮"AI辅助"。点击展开对话入口，可用自然语言修改本体结构（如"给维修工单添加一个紧急程度属性"） |

### 交互细节

**节点拖拽：** 拖拽节点时，与该节点相连的边跟随移动。松开后节点固定在新位置（退出力导向）。

**双击节点：** 进入类属性编辑页（与单击进入相同，双击是快捷方式）。

**画布操作：** 鼠标滚轮缩放（最小50%、最大200%）。按住空白处拖拽平移画布。右上角有缩放控制按钮（+、-、适配全部）。

**新增关系：** 从一个节点拖拽到另一个节点，松开后弹出关系创建面板（输入关系名、选择方向和多重性）。

## 页面 3B：实例图谱浏览器

> 视觉设计稿：`design/ontology_instance_graph_explorer.html`

### 定位

页面三（图谱全貌审核）的实例视图子页面。Schema 视图展示类和关系的抽象结构，实例视图展示**真实数据节点和关系**。让本体设计者验证"我设计的图谱结构在真实数据上是否好用"。

### 与页面三的关系

同一个图谱审核页面，通过顶部 Tab 切换：「结构视图」（已有的 schema 图谱）和「实例视图」（本页面）。共享左侧类列表和顶部导航。

### 五种推演模式详情

| # | 模式 | 起点 | 算法 | 典型场景 |
|---|---|---|---|---|
| 1 | 正向扩散 | 单节点 | BFS N跳展开，沿关系 out 方向 | 设备→备件→头寸：评估故障影响范围 |
| 2 | 反向溯源 | 单节点 | BFS N跳展开，沿关系 in 方向 | 头寸←出入库→设备：追溯消耗原因 |
| 3 | 路径发现 | 双节点 | 最短路径（Dijkstra / allShortestPaths） | 呆滞件→替代→在用件：找消化路径 |
| 4 | 子图聚合 | 类类型 | MATCH (n:Label) 按属性 GROUP BY | 按库房汇总头寸金额/缺口/呆滞数 |
| 5 | 假设推演 | 子图 | 复制子图到内存，修改属性，重算派生 | 安全库存从2→5，缺口怎么变？ |

### 数据来源

| 数据 | 来源 |
|---|---|
| 实例节点和关系 | 图数据服务（通过语义图谱查询 MCP 工具） |
| 节点属性值 | 本体语义标记的属性 |
| 统计数字 | 图查询聚合 |
| 推演结果 | 图路径查询（可变长度路径 / 最短路径） |
| 假设推演 | 前端内存计算（不查询后端） |

### 交互细节

**节点点击：** 单击节点，右侧面板展示节点详情。图谱中该节点放大并加粗边框。

**节点双击：** 以该节点为起点，执行当前选中的推演模式。

**推演动画：** 推演路径从起点开始，逐跳高亮（每跳 300ms 延迟），最终所有路径节点常亮。

**过滤联动：** 关闭某类型后，该类型的节点和相关边从画布移除。力导向布局自动重新计算。

**搜索联动：** 搜索框输入后，匹配节点高亮+居中，不匹配节点降低透明度。清空搜索框恢复。

**数据量保护：** 超过 2000 个节点时，自动按语义标记属性聚合同类型节点（如"二级库-A 的 120 个头寸"合并为一个聚合节点），点击聚合节点展开。

| 6.3 | 问题卡片 | 每个问题一张卡片。包含：问题类型标签 + 问题描述 + 技术细节（等宽字体） + 操作按钮 |
| 6.4 | 自动修复 | 部分问题支持一键自动修复（绿色按钮）。点击后Agent自动修改本体YAML并刷新审核结果 |
| 6.5 | 忽略/标记后续 | 不需要立刻处理的问题可"忽略"或"标记为后续版本"。忽略后该问题不阻塞发布 |
| 6.6 | 通过项列表 | 默认折叠。展开后显示所有通过的检查项，每项前面一个绿色对勾 |
| 6.7 | 重新审核 | 点击后重新运行Agent-S4的全部检查。适用于手动修改本体后重新验证 |
| 6.8 | 发布入口 | "全部处理完毕，发布"按钮。只有当所有一致性问题处理完毕后才可点击（完整性和建议可未处理） |

### 审核检查项目

| 分类 | 检查项 | 严重程度 |
|---|---|---|
| 一致性 | 关系的from/to引用了已定义的类 | 阻断 |
| 一致性 | 派生属性公式中引用的属性存在 | 阻断 |
| 一致性 | 规则条件中引用的属性存在 | 阻断 |
| 一致性 | 动作writes中引用的属性存在 | 阻断 |
| 一致性 | 枚举值使用一致（同一个enum在不同位置的值列表相同） | 阻断 |
| 完整性 | 第一公民有足够的属性（至少10个） | 警告 |
| 完整性 | 无孤立类（每个类至少参与一个关系） | 警告 |
| 完整性 | 每个可写属性至少有一个动作能修改它 | 警告 |
| 完整性 | 每个规则至少有一个触发源 | 警告 |
| 完整性 | 枚举类型的状态字段有完整的状态转换覆盖 | 警告 |
| 命名 | 所有ID使用snake_case | 警告 |
| 命名 | 中文名称无歧义（不同类中同名属性需确认语义一致） | 建议 |
| 一致性 | 指标公式引用的属性和关系存在 | 阻断 |
| 一致性 | 指标 depends_on 引用的指标 ID 存在 | 阻断 |
| 完整性 | 所有 status=implemented 的指标有对应的 tool | 警告 |
| 完整性 | 无 status=undefined 的指标（应至少为 designed） | 建议 |
| 优化 | 无冗余属性（不同类中可合并的属性） | 建议 |
| 优化 | 无冗余关系（可合并的关系） | 建议 |

### 功能清单

| # | 功能 | 说明 |
|---|---|---|
| 7.1 | 版本对比栏 | 顶部显示版本变化：旧版本→新版本+变更摘要（新增了什么、修改了什么） |
| 7.2 | 管道步骤（核心 + 插件） | 按顺序展示管道步骤。核心步骤（MCP工具接口、Agent配置、规则引擎配置、前端类型）+ 可选插件（PG schema、Neo4j schema、连接器映射）。每步是一个时间线节点，包含：步骤名 + 状态标签（完成/运行中/等待/失败）+ 输出代码预览 + 统计数字 |
| 7.3 | 管道步骤详情 | 每步可展开查看完整的生成输出。代码预览区域用等宽字体，高亮新增/修改的部分 |
| 7.4 | 步骤状态 | **完成**（绿色）：正常完成。**完成·建议审核**（绿色+蓝色标签）：完成但有建议需人工确认（如Agent提示词更新建议）。**运行中**（蓝色+动画）：正在执行。**等待**（灰色）：等待上一步完成。**失败**（红色）：生成出错，展示错误信息 |
| 7.5 | Agent配置审核 | Agent配置更新步骤中，如果生成了提示词更新建议，显示建议内容和[确认]/[跳过]按钮。确认后才更新Agent提示词 |
| 7.6 | 总结栏 | 底部显示：完成步骤数/总步骤数 + 总耗时 |
| 7.7 | 变更日志 | 点击"变更日志"展开本次发布的完整变更记录：新增了什么、修改了什么、删除了什么，按类/关系/规则/动作分类 |
| 7.8 | 回滚 | 点击"回滚"恢复到上一个已发布版本。回滚操作会重新运行管道，用旧版本YAML重新生成所有下游代码。需二次确认 |
| 7.9 | 部署到生产 | 点击后将本次发布的代码变更应用到生产环境（执行PG迁移脚本、重启MCP服务、更新Neo4j、刷新Agent配置）。需二次确认 |
| 7.10 | 增量生成 | 管道只生成差异部分。如果本次发布只新增了一个属性，PG生成的是ALTER TABLE而不是全量CREATE TABLE。统计数字清楚展示新增/修改/删除的数量 |

### 管道步骤的输入输出

**核心步骤**

| 步骤 | 输入 | 输出 | 说明 |
|---|---|---|---|
| MCP工具接口 | 本体YAML的classes、actions和metrics | 工具注册JSON（inputSchema/outputSchema） | 每个类→query工具，每个动作→execute工具，每个指标→query/classify工具 |
| Agent配置 | 本体YAML的actions（权限字段） | 每个Agent的工具绑定配置 + 提示词更新建议 | 基于动作权限推导Agent-工具绑定 |
| 规则引擎配置 | 本体YAML的rules | 规则引擎配置文件 | trigger/condition/action翻译为可执行逻辑 |
| 前端类型 | 本体YAML的classes | TypeScript interface定义 | 类→interface，属性→字段 |

**可选插件**

| 步骤 | 输入 | 输出 | 说明 |
|---|---|---|---|
| PG schema 插件 | 本体YAML的classes和relationships | DDL语句（CREATE TABLE / ALTER TABLE） | 可选插件，每个类→一张表，关系→外键 |
| Neo4j schema 插件 | 本体YAML的classes和relationships | 节点标签、关系类型、同步属性清单 | 可选插件，按本体语义结构生成图模型 |
| 连接器映射模板 | 本体YAML的classes属性 | 映射配置骨架 | 新增属性标记为"待映射" |
